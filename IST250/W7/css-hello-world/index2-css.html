Introduction to CSS (con't)

Some key info... Some things to know

1. Where does the code go (INTERNAL css)? Two places:

  • A. < style > code goes in the document < head > section.
  • B. "class" or "id" used in html.

2. Where does the code go (EXTERNAL css)? Two places:

  • A. < style > code goes in a separate file (.css)
    1. < LINK rel="stylesheet" href="my-external-css-file.css" type="text/css">
    2. < LINK rel="stylesheet" media="aural" href="corporate-aural.css" type="text/css">
    3. < LINK rel="stylesheet" media="screen" href="corporate-screen.css" type="text/css">
    4. < LINK rel="stylesheet" media="print" href="corporate-print.css" type="text/css">
  • B. "class" or "id" used in html.

3. What does the code syntax (grammar) look like? Some vocabulary?

  • A. The code snippets in the < head > of your html file OR in the external .css file:
    • Declaration - the code that contains the selector; the whole bit of code that goes in the page < head >.
      • Selector - A “selector” is the instruction in a CSS rule set that tells the browser what elements to ‘select’ for styling.
      • Property - the attribute part of the declaration.
      • Property Value - the value that you assign to the property statement.
  • B. The code snippets in the < body > of your html file:
    • ID (#) = A person's Identification (ID) is unique to one person.
    • Class (.) = There are many people in a class.
